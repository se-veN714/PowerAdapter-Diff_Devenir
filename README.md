# PAdif

给散文 / 长文做 **git 式版本监管 + 句子级差异对比** 的小工具。

代码有 git，写散文的人没有。PAdif 让你像提交代码一样，给一篇文章「提交一版、写句说明、日后挑两版看差异」——而且它**只对比「句子」而不是「行」**，改一个字时只高亮那一个字，不会把整段标红。

---

## 它能帮你什么

- 写长文改了十几遍，想知道「这一版和上一版到底差在哪」——选两版，看字符级高亮。
- 用 Obsidian / VS Code / 任意 md 编辑器写作，希望**在编辑器旁边默默留版**，不用切到另一个写作框。
- 本地留了好几份草稿，想导入进来一起对比。

---

## 快速开始

### 环境要求

- Python 3.10+（只用标准库，无需 `pip install` 任何东西）
- 任意现代浏览器

### 运行

```bash
cd server
python app.py
```

启动后用浏览器打开：

```
http://127.0.0.1:18887/
```

（端口可在启动时用 `PADIF_PORT` 修改，见下方配置。）

---

## 三种使用方式

**1. 导入（新建一篇文章）**
把本地一个 `.md` 文件导入，作为这篇文章的 1.0.0 初始版本。

**2. 监听 + 提交（在你自己的编辑器旁留版）**
填上你正在写的 `.md` 文件绝对路径，点「开始监听」：
- 立即为文件拍一张「初始快照」；
- 之后你照常在自己的编辑器里写，**写完保存、关闭编辑器**时，PAdif 会自动补一版（带时间戳，仅作便利）；
- 更推荐**手动提交**：在页面里点「提交版本」，写一句「这次改了什么」——这句说明是自动提交永远给不了的，也是留给「未来的你」的线索。

> 设计哲学：**自动提交只是兜底，手动提交才是主线。** 工具在旁边默默兜底，但从不替你思考、从不抢你的编辑器。

**3. 添加版本（本地多版本对比）**
选中一篇文章后，可把**另一个本地 `.md` 文件**追加为它的新版本——适合你本地留了多份草稿、想导入一起对比的场景。

---

## 差异视图

提交两版后，选两版即可对比，支持三种视图：
- **行内**：每句独占一行，句内改字仍字符级高亮（「已→它」「不→别」只高亮那一个字）。
- **并排双栏**：左栏显示删除（红），右栏显示新增（绿）。
- **统计摘要**：字数 / 句数 / 段数的变化量。

段落重排（内容相同只是位置变了）会被识别为「移动」，中性渲染，不会虚增红绿噪声。

## 配置

所有配置通过环境变量设置（均在 `server/app.py` 启动前设定）：

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| `PADIF_PORT` | `18887` | 服务监听端口 |
| `PADIF_WATCH_DEBOUNCE` | `30` | 稳定窗口（秒）：文件连续 N 秒无变化才视为「写完了」再自动提交 |
| `PADIF_WATCH_MIN_INTERVAL` | `180` | 最小提交周期（秒）：两次自动提交至少间隔 N 秒 |
| `PADIF_WATCH_CLOSE_PROC` | `Obsidian.exe` | 监听该进程退出即视为「写完关闭」并强制落版；置空则禁用 |

示例（关闭编辑器即提交，改较短的稳定窗口）：

```bash
PADIF_WATCH_DEBOUNCE=10 PADIF_WATCH_CLOSE_PROC="Code.exe" python app.py
```

## 数据说明

所有版本数据存在单个 SQLite 文件 `data/padif.db`，纯本地、**无云服务依赖**。删除该文件即清空全部历史。

## 目录结构

```
padif/
├── server/          # 后端（标准库 http.server，零依赖）
│   ├── app.py       # 服务入口 / 路由编排
│   ├── differ.py    # 句子级 diff 引擎
│   ├── version.py   # 语义版本引导（major/minor/patch）
│   ├── store.py     # SQLite 存储
│   └── watcher.py   # 文件监听守护线程
├── web/             # 前端（htmx + 原生 JS，htmx.min.js 本地自带）
│   ├── index.html
│   ├── app.js
│   └── htmx.min.js
├── samples/         # 示例 / 测试用 md
└── data/           # padif.db（运行时生成）
```
