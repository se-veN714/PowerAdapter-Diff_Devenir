# 氧潮手记：PAdif 的诞生

PAdif 是我们为深度内容创作者打造的版本监管工具。它不似传统版本控制那般为代码而生，而是专为散文与长文设计。创作者每完成一段有意义的修改，便能像做一次 commit 那样，为文章留下一个带语义版本的快照。它纯粹运行在本地，无需任何云端账号。

我们最初面对的核心痛点是：散文的修改是连续的、不分行的。普通的文本 diff 会把整段标红，根本看不出究竟改了哪几个字。于是我们决定以「句子」作为最小单位，让每一版之间的差异精确到标点与字词。

```python
# 启动本地版本监管服务
from server.app import main
main()  # 默认监听 18887 端口，可用 PADIF_PORT 覆盖
```

## 设计上的几个选择

- 提交只引导、不拦截：我们信任创作者有自己的小巧思，工具仅作辅助标注。
- 先网页、后插件：第一阶段交付本地网页应用，未来再迭代为 Obsidian 插件。
- 轻量优先：不引入重型依赖，本地存储、单机使用、零云依赖。
- 差异可读：同时支持行内、并排与统计三种视图。

截至目前，我们已经跑通了「导入 Markdown、提交语义版本、选两版查看句子级差异」的完整链路。接下来，我们打算让工具自动监听文件变化，把手动提交的成本继续降下来。自动提交尚属候选方向，创作者依旧可完全手动掌控节奏。
